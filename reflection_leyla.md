During this phase of the RecipeLog project, my primary contribution was focused on the visual consistency and user experience of the application. I took on the task of auditing the navigation bar and authentication styles to ensure that all interactive elements felt like part of a unified system. Specifically, I worked on transforming what were originally plain text links—such as "Sign Up," "Home," and "History"—into polished, clickable buttons that perfectly match the "Log In" action. My goal was to make the interface feel intuitive and professional, providing clear visual cues for every primary action available to the user.

The biggest technical challenge I faced was the strict requirement not to touch the underlying HTML logic or structure. I had to achieve the entire redesign using CSS alone. This forced me to get creative with advanced selectors; for instance, I used attribute matching to target specific routes and the :not() pseudo-class to apply styles without being able to add new classes to the tags. It felt like solving a complex puzzle, but successfully styling the site within these constraints was incredibly rewarding. I am most proud of how clean the UI looks now; it has transitioned from a basic assignment into something that feels like a real-world product. If I could redo one thing, I would probably organize my CSS variables more efficiently from the start to avoid some of the redundant code I wrote while targeting each button individually.

Our team collaboration was solid, and we maintained a steady workflow using Git and GitHub. We followed a feature-branch system, which was a great learning experience, even though I initially had some "Git panic" moments while trying to remember the commands for switching and publishing branches. From my teammates, I learned a lot about maintaining a clean codebase and the importance of testing UI changes before merging them into the main branch. In the final project, I plan to suggest a shared style guide at the very beginning to prevent our CSS from clashing.

This project has been a huge growth opportunity for me. I’ve significantly improved my skills with CSS Flexbox and terminal-based Git operations. AI also played a helpful role, acting as a "pair programmer" when I needed to brainstorm specific selectors or get a quick reminder of Git syntax. Looking ahead to the final project, I am excited to explore Real-Time Updates. I want to learn how to implement features like live notifications to make the app feel more dynamic and modern without constant page refreshing.
